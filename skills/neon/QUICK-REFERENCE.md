## Environment Setup

```bash
# Required in ~/shared/.env
NEON_API_KEY=neon_api_xxx                # Management API
NEON_DATABASE_URL=postgresql://...       # Direct SQL
NEON_DATA_API_URL=https://ep-xxx...     # Data API
NEON_JWT_TOKEN=eyJhbGci...              # Data API auth
```

---

## Data API (REST)

### Query (GET)
```bash
# Basic
curl "$NEON_DATA_API_URL/posts" -H "Authorization: Bearer $JWT"

# With filters
curl "$NEON_DATA_API_URL/posts?is_published=eq.true&order=created_at.desc&limit=10" -H "Authorization: Bearer $JWT"
```

### Insert (POST)
```bash
curl -X POST "$NEON_DATA_API_URL/posts" \
  -H "Authorization: Bearer $JWT" \
  -H "Content-Type: application/json" \
  -d '{"content": "Hello", "is_published": false}'
```

### Update (PATCH)
```bash
curl -X PATCH "$NEON_DATA_API_URL/posts?id=eq.1" \
  -H "Authorization: Bearer $JWT" \
  -H "Content-Type: application/json" \
  -d '{"is_published": true}'
```

### Delete
```bash
curl -X DELETE "$NEON_DATA_API_URL/posts?id=eq.1" \
  -H "Authorization: Bearer $JWT"
```

### Filters
```
?column=eq.value      # equals
?column=neq.value     # not equals
?column=gt.10         # greater than
?column=lt.10         # less than
?column=like.*hello*  # pattern match
?column=in.(a,b,c)    # in list
```

## Direct SQL (Python)

```python
import psycopg2
import os

conn = psycopg2.connect(os.getenv('NEON_DATABASE_URL'))
cur = conn.cursor()

# Query
cur.execute("SELECT * FROM posts WHERE is_published = %s", (True,))
rows = cur.fetchall()

# Insert
cur.execute("INSERT INTO posts (content) VALUES (%s) RETURNING id", ("Hello",))
post_id = cur.fetchone()[0]
conn.commit()

cur.close()
conn.close()
```

## Management API

### List Projects
```bash
curl "https://console.neon.tech/api/v2/projects" \
  -H "Authorization: Bearer $NEON_API_KEY"
```

### Create Branch
```bash
curl -X POST "https://console.neon.tech/api/v2/projects/$PROJECT_ID/branches" \
  -H "Authorization: Bearer $NEON_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"branch": {"name": "dev"}, "endpoints": [{"type": "read_write"}]}'
```

### Get Connection String
```bash
curl "https://console.neon.tech/api/v2/projects/$PROJECT_ID/connection_uri?branch_id=$BRANCH_ID" \
  -H "Authorization: Bearer $NEON_API_KEY"
```

## Row-Level Security (RLS)

### Enable RLS
```sql
ALTER TABLE posts ENABLE ROW LEVEL SECURITY;
```

### Basic Policy (Own Data)
```sql
CREATE POLICY manage_own ON posts
FOR ALL TO authenticated
USING (auth.user_id() = user_id)
WITH CHECK (auth.user_id() = user_id);
```

### Public Read Policy
```sql
CREATE POLICY read_all ON posts
FOR SELECT TO authenticated
USING (true);
```

### Test Policy
```sql
SET LOCAL request.jwt.claims TO '{"sub": "user-123"}';
SELECT * FROM posts;
RESET request.jwt.claims;
```

## Common Operations

### Create Table with RLS
```sql
CREATE TABLE posts (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id text DEFAULT (auth.user_id()) NOT NULL,
  content text NOT NULL,
  created_at timestamptz DEFAULT now() NOT NULL
);

ALTER TABLE posts ENABLE ROW LEVEL SECURITY;

CREATE POLICY manage_own ON posts FOR ALL TO authenticated
USING (auth.user_id() = user_id) WITH CHECK (auth.user_id() = user_id);

-- Refresh schema cache!
```

### Refresh Schema Cache
```bash
curl -X PATCH "https://console.neon.tech/api/v2/projects/$PROJECT_ID/branches/$BRANCH_ID/data-api/$DATABASE" \
  -H "Authorization: Bearer $NEON_API_KEY" -d '{}'
```

### Connection Pool
```python
from psycopg2 import pool

pool = pool.SimpleConnectionPool(1, 10, dsn=DATABASE_URL)
conn = pool.getconn()
# ... use conn ...
pool.putconn(conn)
```

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Schema not updating | Refresh schema cache |
| RLS blocking query | Check policies with `SELECT * FROM pg_policies` |
| Rate limited | Implement exponential backoff |
| JWT expired | Get new token (expires ~15 min) |
| Connection refused | Check connection string format |

---

## Rate Limits

- Management API: ~100/min
- Data API: Compute-dependent
- SQL: Connection pool limits

## Remember

1. ✅ Always enable RLS on user tables
2. ✅ Use parameterized queries (`%s`, not f-strings)
3. ✅ Refresh schema cache after changes
4. ✅ Test RLS policies thoroughly
5. ❌ Never commit credentials
6. ❌ Never use string interpolation for SQL
