# LINKING.md — Links, Backlinks, Embeds, and Graph Reference

## 1. Link Types — All Syntax Variants

### Wikilinks (internal)

| Syntax | Output | Use |
|--------|--------|-----|
| `[[Note Title]]` | Link to note | Standard internal link |
| `[[Note Title\|Display Text]]` | Link with alias | When note title is too long or unclear in context |
| `[[Note#Heading]]` | Link to heading | Jump directly to a section |
| `[[Note#^blockID]]` | Link to block | Jump to a specific paragraph |
| `[[Folder/Note Title]]` | Link with path | When note names conflict across folders |

### Embeds (inline rendering)

| Syntax | Output |
|--------|--------|
| `![[Note Title]]` | Renders the full note inline |
| `![[Note#Heading]]` | Renders from that heading to the next |
| `![[Note#^blockID]]` | Renders just that paragraph/block |
| `![[image.png]]` | Renders an image |
| `![[file.pdf]]` | Embeds a PDF viewer |

Embeds are powerful for dashboards and hub notes — embed a task list or project status block without leaving the note.

### Markdown links

| Syntax | Use |
|--------|-----|
| `[Text](note.md)` | Portable markdown — works outside Obsidian |
| `[Text](https://url.com)` | External URL |
| `[Text](obsidian://open?vault=Name&file=Path)` | Deep link to a specific vault/file |

**Prefer wikilinks for internal notes.** Use markdown links only when the file will be read outside Obsidian or when linking externally.

## 2. Block References

Add a block ID to any paragraph to make it linkable:

```markdown
This is an important paragraph I want to reference from elsewhere. ^my-block-id
```

Then link to it from another note:
```markdown
See [[Note Title#^my-block-id]] for the key insight here.
```

Or embed it:
```markdown
![[Note Title#^my-block-id]]
```

**Block IDs:** lowercase, hyphens only, no spaces. Auto-generated by Obsidian when you use "Copy link to block" — looks like `^abc123`. You can rename them to something readable.

**When to use block references:** when you want to cite or display a specific sentence or paragraph, not the whole note or section.

## 3. Backlinks Panel

The Backlinks panel (right sidebar) shows every note that links to the current note.

**What to use it for:**
- Find which notes reference the current one — understand context and usage
- Identify orphaned notes (notes with no backlinks are isolated; either link them or consider deleting)
- Discover unexpected connections

**Linked mentions:** notes with explicit wikilinks to the current note.
**Unlinked mentions:** notes that contain the text of the current note's title but don't link to it. Obsidian finds these automatically — click to create the link.

---

## 4. Unlinked Mentions

Obsidian scans all notes for text that matches note titles but isn't linked. These appear in the Backlinks panel under "Unlinked mentions."

**Workflow:** periodically review unlinked mentions for your most important notes. If the mention is meaningful, click to convert it to a wikilink. If it's a coincidental match (e.g. a common word), ignore it.

This is one of the fastest ways to strengthen the graph without extra effort.

## 5. Graph View Strategies

Graph view shows notes as nodes and links as edges.

### Configuration

| Setting | What to do |
|---------|-----------|
| Groups | Color-code by type: MOCs = blue, daily = gray, permanent = green |
| Filters | Hide `#type/daily` notes to reduce clutter; show only `#status/active` |
| Local graph | Cmd/Ctrl + click a note → see only its direct connections |
| Orphans | Filter to show only notes with no links → find notes that need connecting |

### Local graph (most useful)

Open from the note header (three-dot menu → Open local graph) or Cmd/Ctrl + click in main graph. Shows the note's direct neighbors up to a configurable depth. Use this to navigate a topic cluster.

### Graph as health indicator

A healthy vault graph has:
- Most notes connected to 3+ others
- Visible clusters around MOCs and hub notes
- Few isolated nodes (orphans)

An unhealthy graph has many isolated nodes — notes that were captured but never integrated into the knowledge network. Monthly review should address these.

## 6. Obsidian Advanced URI

The Advanced URI plugin (installed in Yasmine-OS) creates links that trigger specific Obsidian actions from outside the app — from automations, browsers, or other apps.

### Common patterns

| Action | URI |
|--------|-----|
| Open a note | `obsidian://advanced-uri?vault=Yasmine-OS&filepath=path/to/note.md` |
| Open and go to heading | `obsidian://advanced-uri?vault=Yasmine-OS&filepath=note.md&heading=Section` |
| Create a new note | `obsidian://advanced-uri?vault=Yasmine-OS&filepath=new-note.md&newpane=true` |
| Run a command | `obsidian://advanced-uri?vault=Yasmine-OS&commandid=command-id` |

**Use case for Yasmine:** automation scripts (e.g. `python create_note.py`) can build these URIs and open the correct note in Obsidian without the REST API. Also useful for linking from Notion, browser bookmarks, or Claude outputs.

## 7. Link Maintenance — Finding Broken Links

**Outgoing Links panel** (right sidebar): shows all wikilinks in the current note. Unresolved links (pointing to notes that don't exist) appear in red or in a separate "Unresolved" section.

**Fixing broken links:**
1. The target note was renamed — update the link, or rename via Obsidian (auto-updates all links if "Automatically update internal links" is on in Settings → Files & Links)
2. The target note was deleted — remove the link or recreate the note
3. Typo in the link — correct it

**Settings → Files & Links → Automatically update internal links:** turn this ON. It ensures that renaming or moving a note updates all links pointing to it.

## 8. Linking Philosophy for Yasmine

Link when the connection is **meaningful**, not just when words match.

**Good reasons to link:**
- The linked note provides evidence, context, or elaboration for the current note
- Following the link would genuinely help someone understand the current note
- The two notes address the same idea from different angles

**Avoid linking:**
- Just because a word matches a note title (use unlinked mentions review instead of reflexive linking)
- Creating links as a substitute for actually integrating ideas
- Linking every mention of "AI" to a generic AI overview note

**Target density:** 3–7 outgoing links per permanent note. Fewer = isolated. More = probably linking indiscriminately.
