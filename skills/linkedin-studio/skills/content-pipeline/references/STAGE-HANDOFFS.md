# Stage Handoffs Reference

## Overview

Each pipeline stage receives a typed input contract and emits a typed output contract. The output of one stage is the input of the next. No stage may skip or omit required fields.

---

## INPUT → RESEARCH

```json
{
  "niche": "string",
  "keywords": ["string"],
  "content_pillars": ["string"],
  "time_range": "string",
  "max_topics": "integer"
}
```

**Schema**

| Field           | Type            | Required | Description                                         |
|-----------------|-----------------|----------|-----------------------------------------------------|
| niche           | string          | Yes      | Industry or topic domain (e.g. "SaaS growth")       |
| keywords        | array[string]   | Yes      | Seed keywords to guide topic discovery              |
| content_pillars | array[string]   | Yes      | Strategic content categories (e.g. "thought leadership") |
| time_range      | string          | Yes      | Recency filter (e.g. "7d", "30d")                   |
| max_topics      | integer         | Yes      | Maximum topics the research stage should return     |

---

## RESEARCH → WRITE

```json
{
  "topic_id": "string",
  "raw_topic": "string",
  "research_angles": ["string"],
  "competitor_gaps": ["string"],
  "content_pillar": "string",
  "hook_suggestions": ["string"],
  "cta_suggestions": ["string"],
  "strategy_context": {
    "relevant_stories": ["string"],
    "audience_segment": "string",
    "positioning_angle": "string",
    "voice_markers": ["string"]
  }
}
```

**Schema**

| Field             | Type          | Required | Description                                          |
|-------------------|---------------|----------|------------------------------------------------------|
| topic_id          | string        | Yes      | Unique identifier for this topic (generated by RESEARCH) |
| raw_topic         | string        | Yes      | Topic title or headline as identified                |
| research_angles   | array[string] | Yes      | Distinct angles or framings for the topic            |
| competitor_gaps   | array[string] | Yes      | Gaps in existing competitor content to exploit       |
| content_pillar    | string        | Yes      | Assigned content pillar from `content_pillars`       |
| hook_suggestions  | array[string] | Yes      | Candidate hook lines generated during research       |
| cta_suggestions   | array[string] | Yes      | Candidate CTA phrases relevant to the topic          |
| strategy_context    | object        | Yes      | Strategy context loaded from strategy/ directory         |
| ├─ relevant_stories | array[string] | Yes      | Story IDs from STORY-BANK.md matching topic (may be empty) |
| ├─ audience_segment | string        | Yes      | ICP segment from AUDIENCE.md                             |
| ├─ positioning_angle| string        | Yes      | Competitive moat angle from POSITIONING.md               |
| └─ voice_markers    | array[string] | Yes      | Yasmine voice markers relevant to this post              |

---

## WRITE → HUMANIZE

```json
{
  "post_text": "string",
  "hook": "string",
  "content_pillar": "string",
  "post_id": "string",
  "post_type": "string",
  "brand_voice_profile": {
    "tone": "string",
    "vocabulary": ["string"],
    "avoid_words": ["string"]
  }
}
```

**Schema**

| Field               | Type    | Required | Description                                             |
|---------------------|---------|----------|---------------------------------------------------------|
| post_text           | string  | Yes      | Full drafted post body (passes G-W1 and G-W2)          |
| hook                | string  | Yes      | Extracted first line / hook sentence                   |
| content_pillar      | string  | Yes      | Assigned pillar (carried forward from RESEARCH)        |
| post_id             | string  | Yes      | Unique post identifier (generated at WRITE stage)      |
| post_type           | string  | Yes      | `"standard"`, `"carousel"`, `"article"`, or `"poll"`   |
| brand_voice_profile | object  | Yes      | Brand voice config used to guide humanization          |

---

## HUMANIZE → STRUCTURE

```json
{
  "humanized_content": "string",
  "post_id": "string",
  "ai_score": "number",
  "brand_voice_profile": {
    "tone": "string",
    "vocabulary": ["string"],
    "avoid_words": ["string"]
  }
}
```

**Schema**

| Field               | Type   | Required | Description                                                |
|---------------------|--------|----------|------------------------------------------------------------|
| humanized_content   | string | Yes      | Post text after humanization pass                         |
| post_id             | string | Yes      | Same post_id carried forward                              |
| ai_score            | number | Yes      | AI-detection score AFTER humanization (lower is better)   |
| brand_voice_profile | object | Yes      | Passed through unchanged for STRUCTURE to reference       |

---

## STRUCTURE → VISUAL

```json
{
  "reviewed_content": "string",
  "hook": "string",
  "key_points": ["string"],
  "content_pillar": "string",
  "post_id": "string",
  "quality_score": "integer",
  "hook_score": "integer"
}
```

**Schema**

| Field            | Type          | Required | Description                                             |
|------------------|---------------|----------|---------------------------------------------------------|
| reviewed_content | string        | Yes      | Final reviewed and edited post body                    |
| hook             | string        | Yes      | Finalized hook line after structural review            |
| key_points       | array[string] | Yes      | 3–5 bullet-level key points extracted from content     |
| content_pillar   | string        | Yes      | Carried forward                                        |
| post_id          | string        | Yes      | Carried forward                                        |
| quality_score    | integer       | Yes      | 0–100 quality score assigned during STRUCTURE stage    |
| hook_score       | integer       | Yes      | 0–10 hook strength score assigned during STRUCTURE     |

---

## VISUAL → CALENDAR

```json
{
  "final_content": "string",
  "media_urls": ["string"],
  "ai_prompts": ["string"],
  "post_id": "string",
  "visual_type": "string"
}
```

**Schema**

| Field         | Type          | Required | Description                                                 |
|---------------|---------------|----------|-------------------------------------------------------------|
| final_content | string        | Yes      | Post body — unchanged from STRUCTURE output                |
| media_urls    | array[string] | Yes      | Direct URLs to generated or uploaded assets (may be empty if Canva used) |
| ai_prompts    | array[string] | Yes      | Prompts used for image generation (for audit trail)        |
| post_id       | string        | Yes      | Carried forward                                            |
| visual_type   | string        | Yes      | `"generated"`, `"uploaded"`, `"canva"`, or `"none"`       |

---

## CALENDAR → SCHEDULE

```json
{
  "post_id": "string",
  "scheduled_at": "string",
  "content_pillar": "string"
}
```

**Schema**

| Field          | Type   | Required | Description                                               |
|----------------|--------|----------|-----------------------------------------------------------|
| post_id        | string | Yes      | Carried forward                                          |
| scheduled_at   | string | Yes      | ISO 8601 datetime chosen by the CALENDAR stage           |
| content_pillar | string | Yes      | Used by SCHEDULE to apply pillar-based posting rules     |
